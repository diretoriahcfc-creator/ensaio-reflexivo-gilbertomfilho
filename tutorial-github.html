Tutorial

Publicar o site
no GitHub Pages

Dois caminhos: pela interface web do GitHub (sem instalar nada) ou pelo terminal com Git.

Instalação do Git

1

Baixar o Git para Windows

Acesse o site oficial pelo browser. O download do instalador começa automaticamente assim que a página carrega.

endereço no browser
git-scm.com/download/win
Conta GitHub: você já tem conta criada — o Git que vai instalar aqui é apenas o programa local que permite enviar ficheiros para o GitHub pelo terminal. São coisas separadas.
2

Executar o instalador

Abra o ficheiro .exe que foi descarregado. O instalador vai mostrar várias telas de opções — pode clicar em Next em todas sem alterar nada. As configurações padrão funcionam perfeitamente para o que precisamos.

⚙️
Única atenção durante a instalação Na tela "Adjusting your PATH environment", deixe selecionado "Git from the command line and also from 3rd-party software" (já vem selecionado por padrão).
3

Confirmar a instalação

Após instalar, abra o PowerShell — barra de pesquisa do Windows, escreva powershell e pressione Enter. Em seguida, digite:

PowerShell
git --version

Se aparecer algo como git version 2.x.x, a instalação foi bem-sucedida. Pode avançar para o Método 2 abaixo.

Não apareceu a versão? Feche o PowerShell, abra novamente e tente outra vez. O terminal precisa ser reiniciado para reconhecer o Git recém-instalado.
4

Identificar-se ao Git (uma vez só)

Antes do primeiro uso, diga ao Git quem é você. Use o mesmo e-mail da sua conta GitHub. Estes dois comandos só precisam ser executados uma única vez em todo o computador.

PowerShell — executar uma vez
git config --global user.name "Gilberto Mendonça Filho" git config --global user.email "diretoria.hcfc@gmail.com"
Git instalado — escolha como publicar
1

Aceder ao repositório no GitHub

Abra o browser e entre em github.com. Faça login na sua conta e clique no repositório do Ensaio Reflexivo — normalmente aparece na lista da página inicial ou em Your repositories.

🔗
URL do repositório github.com/seu-usuario/ensaio-reflexivo (ou o nome que escolheu)
2

Fazer upload do index.html actualizado

Dentro do repositório, clique em Add file → Upload files (botão no canto direito, acima da lista de ficheiros). Arraste o ficheiro index.html para a área de drop, ou clique em "choose your files" para selecionar.

Atenção: se o repositório já tem um index.html, o upload substitui automaticamente o ficheiro existente. Não é preciso apagar antes.
📂
Ficheiro a enviar index.html — o que acabou de ser gerado, na pasta do blog no seu computador.
3

Confirmar o commit

Após selecionar o ficheiro, role a página até à secção Commit changes. No campo de texto já vem preenchido algo como "Upload files". Pode deixar assim ou escrever uma descrição mais clara, por exemplo:

mensagem de commit
Atualiza index.html — ancoragem do menu e seção Fontes

Deixe selecionado "Commit directly to the main branch" e clique em Commit changes.

4

Aguardar a publicação (1 a 3 minutos)

O GitHub Pages detecta o commit e publica automaticamente. Para acompanhar: clique em Settings → Pages no menu do repositório. Quando estiver pronto, aparece a mensagem em verde: "Your site is live at …"

Dica: se o site ainda mostrar a versão antiga, force o recarregamento no browser com Ctrl + Shift + R (Windows) ou Cmd + Shift + R (Mac). O browser pode ter em cache a versão anterior.
1

Pré-requisito: Git instalado

Abra o terminal (PowerShell no Windows, Terminal no Mac) e confirme que o Git está instalado com o comando abaixo. Se devolver a versão, está tudo certo.

terminal
git --version

Se aparecer "git is not recognized" ou semelhante, aceda a git-scm.com, faça o download e instale. Depois reinicie o terminal.

2

Navegar até à pasta do blog

No terminal, entre na pasta onde estão os ficheiros do blog. Substitua o caminho pelo caminho real no seu computador.

Windows — PowerShell
cd "C:\Users\Jose Gilberto\Documents\Pasta Gilberto\Gilberto Filho\Blog\ensaio reflexivo"
Mac — Terminal
cd ~/Documents/ensaio-reflexivo
3

Verificar o estado dos ficheiros

Este comando mostra quais ficheiros foram alterados desde o último commit. O index.html deve aparecer na lista.

terminal
git status
4

Adicionar, confirmar e enviar

Três comandos em sequência: add prepara o ficheiro, commit regista a alteração localmente, push envia para o GitHub.

terminal — executar na ordem
git add index.html git commit -m "Atualiza index.html — ancoragem do menu e seção Fontes" git push origin main
Nota: se o branch principal do seu repositório se chamar master em vez de main, substitua git push origin main por git push origin master.

O terminal pode pedir o utilizador e a senha (ou um token de acesso pessoal) do GitHub na primeira vez. Isso é normal e não acontece nas vezes seguintes se optar por guardar as credenciais.

5

Confirmar a publicação

Após o push, o GitHub Pages publica em 1 a 3 minutos. Aceda ao seu site e faça Ctrl + Shift + R para garantir que o browser não usa cache da versão anterior.

Para próximas atualizações — apenas os três comandos são necessários: git add, git commit -m "mensagem", git push origin main.
resultado esperado
